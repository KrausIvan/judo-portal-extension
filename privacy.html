Zásady ochrany soukromí

Rozšíření Judo rozhodčí → Kalendář
Poslední aktualizace: 14. dubna 2026

1. Kdo provozuje rozšíření

Toto Chrome rozšíření vyvíjí a spravuje Ivan Kraus jako neziskový osobní projekt pro komunitu rozhodčích Českého svazu judo. Není provozováno jménem ČSJU ani žádné jiné organizace.

Kontakt: ivan.kraus04@gmail.com

2. Co rozšíření dělá

Rozšíření načítá seznam delegovaných turnajů ze stránky rozhodci.csju.cz, na které je uživatel přihlášen, a umožňuje tyto turnaje uložit do kalendáře — buď přímo do Google Kalendáře přes oficiální Google Calendar API, nebo jako stažený .ics soubor pro libovolnou kalendářovou aplikaci.

3. Jaká data rozšíření zpracovává

3.1 Data z portálu rozhodčích

Rozšíření čte z aktuálně otevřené stránky rozhodci.csju.cz následující informace o turnajích, ke kterým má přihlášený uživatel přístup:

  • Název turnaje, datum, místo konání
  • Role uživatele (Rozhodčí / Technický)

Tato data se zpracovávají výhradně lokálně v prohlížeči uživatele. Neukládají se na žádný server, neodesílají se třetím stranám a nejsou nijak agregována ani analyzována.

3.2 Údaje o uživatelově Google účtu

Pokud se uživatel dobrovolně rozhodne použít přímou integraci s Google Kalendářem, rozšíření si od Googlu prostřednictvím protokolu OAuth 2.0 vyžádá oprávnění k těmto datům:

  • E-mailová adresa přihlášeného Google účtu (scope userinfo.email) — zobrazuje se v rozhraní rozšíření, aby uživatel viděl, do kterého účtu události zapisuje. E-mail se nikam neukládá, nepřenáší a neopouští zařízení uživatele.
  • Zápis událostí do Google Kalendáře (scope calendar.events) — používá se výhradně k vytváření nových událostí odpovídajících turnajům. Rozšíření nečte, neupravuje ani nemaže existující události uživatele.

OAuth token je uložen pouze v zabezpečeném úložišti prohlížeče Chrome (chrome.identity) a nikdy se nepřenáší jinam než do oficiálních API Googlu.

3.3 Uživatelská nastavení

Předvolby uživatele (zvolená barva událostí, časy připomínek, cílový kalendář) se ukládají lokálně v prohlížeči přes chrome.storage. Tato data zůstávají v zařízení uživatele a synchronizují se pouze v rámci jeho vlastní Chrome synchronizace, pokud ji má zapnutou.

4. Sdílení dat se třetími stranami

Žádná data uživatele nejsou sdílena s žádnou třetí stranou. Rozšíření komunikuje pouze s těmito službami:

  • rozhodci.csju.cz — pouze čtení obsahu stránky, kterou si uživatel sám otevřel
  • googleapis.com — pouze pokud uživatel aktivně použije funkci „Uložit do Google Kalendáře"

Rozšíření neobsahuje žádné analytické nástroje, trackery, reklamní SDK ani telemetrii.

5. Uchování dat

Rozšíření žádná data dlouhodobě neuchovává mimo zařízení uživatele. Události vytvořené v Google Kalendáři zůstávají v Google Kalendáři uživatele a podléhají zásadám ochrany soukromí Googlu. Uživatel je může kdykoli ručně smazat.

OAuth oprávnění může uživatel kdykoli odvolat na stránce myaccount.google.com/permissions.

6. Práva uživatele

Vzhledem k tomu, že rozšíření žádná data uživatele nesbírá ani neukládá na server, nemá provozovatel k dispozici žádné osobní údaje, které by mohl poskytnout, opravit nebo smazat. Veškerá data jsou v rukou uživatele:

  • Lokální nastavení odstraní odinstalování rozšíření.
  • OAuth token se zruší odvoláním přístupu na myaccount.google.com/permissions.
  • Vytvořené události lze smazat přímo v Google Kalendáři.

7. Soulad s Google API Services User Data Policy

Rozšíření používá Google API Services a dodržuje Google API Services User Data Policy, včetně požadavků na omezené použití dat (Limited Use). Uživatelská data získaná přes Google API Services nejsou:

  • Používána k personalizované reklamě
  • Prodávána ani předávána třetím stranám
  • Používána k jinému účelu, než pro který si uživatel udělil oprávnění
  • Čtena lidskými operátory s výjimkou případů, kdy s tím uživatel výslovně souhlasí, nebo pokud to vyžaduje bezpečnost či právní povinnosti

8. Změny těchto zásad

Tyto zásady se mohou v budoucnu měnit. Aktuální verze je vždy dostupná na této stránce a datum poslední aktualizace je uvedeno v záhlaví dokumentu.

9. Kontakt

Privacy Policy

Browser extension Judo rozhodčí → Kalendář (Judo Referees → Calendar)
Last updated: April 14, 2026

1. Who operates this extension

This Chrome extension is developed and maintained by Ivan Kraus as a non-commercial personal project for the referee community of the Czech Judo Federation. It is not operated on behalf of ČSJU or any other organization.

Contact: ivan.kraus04@gmail.com

2. What the extension does

The extension reads the list of delegated tournaments from the page rozhodci.csju.cz the user is logged into, and lets the user save those tournaments to a calendar — either directly to Google Calendar via the official Google Calendar API, or as a downloaded .ics file usable in any calendar application.

3. What data the extension processes

3.1 Data from the referee portal

The extension reads the following information from the currently open rozhodci.csju.cz page, which the logged-in user already has access to:

  • Tournament name, date, location
  • The user's role (Referee / Technical)

This data is processed exclusively locally in the user's browser. It is never stored on any server, never transmitted to any third party, and never aggregated or analyzed.

3.2 Google account information

If the user voluntarily chooses to use the direct Google Calendar integration, the extension requests the following permissions from Google via OAuth 2.0:

  • Email address of the signed-in Google account (userinfo.email scope) — displayed in the extension UI so the user can verify which account events will be written to. The email is never stored, transmitted, or sent off the user's device.
  • Write access to Google Calendar events (calendar.events scope) — used solely to create new events corresponding to delegated tournaments. The extension does not read, modify, or delete any of the user's existing events.

The OAuth token is stored only in Chrome's secure identity storage (chrome.identity) and is never transmitted anywhere except to Google's official APIs.

3.3 User preferences

User preferences (chosen event color, reminder times, target calendar) are stored locally via chrome.storage. This data stays on the user's device and is only synced through Chrome's own sync feature if the user has it enabled.

4. Sharing data with third parties

No user data is shared with any third party. The extension only communicates with these services:

  • rozhodci.csju.cz — only reads content of pages the user has opened themselves
  • googleapis.com — only when the user actively triggers the "Save to Google Calendar" feature

The extension contains no analytics tools, trackers, advertising SDKs, or telemetry of any kind.

5. Data retention

The extension does not retain any data outside the user's device. Events created in Google Calendar remain in the user's own Google Calendar and are subject to Google's privacy policy. The user can delete them manually at any time.

The user can revoke OAuth permissions at any time at myaccount.google.com/permissions.

6. User rights

Because the extension does not collect or store any user data on a server, the operator does not hold any personal data that could be provided, corrected, or deleted. All data is in the user's hands:

  • Local preferences are removed by uninstalling the extension.
  • The OAuth token is revoked at myaccount.google.com/permissions.
  • Created events can be deleted directly in Google Calendar.

7. Compliance with Google API Services User Data Policy

This extension's use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements. User data obtained via Google API Services is not:

  • Used for personalized advertising
  • Sold or transferred to third parties
  • Used for any purpose other than the one for which the user granted permission
  • Read by human operators except where the user has explicitly consented, or where required for security or legal compliance

8. Changes to this policy

This policy may change in the future. The current version is always available at this page, and the date of the last update is shown in the header of the document.

9. Contact

Questions, requests, or concerns regarding privacy: ivan.kraus04@gmail.com
